# Middleware, CORS & SQL Databases

## Middleware

A **middleware** is a function that is executed with every request before the request is processed by a specific path operation. It can also be used to work with our response from a particular path operation before returning it.

### Creating a Middleware

Use the decorator `@app.middleware("http")` on top of a function.

A middleware function receives:
- The **request**
- A function **`call_next`** that will receive the request as a parameter, pass the request to the corresponding path operation, then return the response generated by that path operation
- You can then further modify the **response** before returning it

```python
import time
from fastapi import FastAPI, Request

app = FastAPI()

@app.middleware("http")  # 🔄 Runs before and after every HTTP request
async def add_process_time_header(request: Request, call_next):
    start_time = time.perf_counter()
    response = await call_next(request)  # 🔄 Pass request to next handler
    process_time = time.perf_counter() - start_time
    response.headers["X-Process-Time"] = str(process_time)  # 🔄 Modify response globally
    return response  # 🔄 Return response after middleware processing
```

### Multiple Middleware Execution

We can add multiple middlewares using either `@app.middleware()` or `app.add_middleware()`. They will be executed in a **stack** manner.

## CORS (Cross Origin Resource Sharing)

- **Origin** — combination of protocol (`http`, `https`), domain (`myapp.com`, `localhost`), and port (`80`, `443`, `8080`)

**Example:** If we have a frontend at `localhost:8080` and backend at `localhost:80`, the browser will send an HTTP OPTIONS request to `:80`. If the backend sends appropriate headers authorizing communication from origin `:8080`, the browser will allow the frontend to make requests to the backend.

To achieve this, the `:80` backend must have a list of "allowed origins".

### CORSMiddleware

1. Import `CORSMiddleware`
2. Create a list of allowed origins (as strings)
3. Add it as a "middleware" to your FastAPI application

```python
from fastapi import FastAPI
from fastapi.middleware.cors import CORSMiddleware

app = FastAPI()

origins = [
    "http://localhost.tiangolo.com",
    "https://localhost.tiangolo.com",
    "http://localhost",
    "http://localhost:8080",
]

app.add_middleware(
    CORSMiddleware,
    allow_origins=origins,
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)

@app.get("/")
async def main():
    return {"message": "Hello World"}
```

Default parameters used by `CORSMiddleware` are **restrictive by default**, so you'll need to explicitly enable particular origins, methods, or headers.

**Arguments supported:**

| Argument | Description |
|---|---|
| `allow_origins` | List of allowed domains (use `['*']` for all origins) |
| `allow_origin_regex` | Regex pattern to match and allow multiple origins dynamically |
| `allow_methods` | List of allowed HTTP methods (default `['GET']`, use `['*']` for all) |
| `allow_headers` | List of allowed request headers (default `[]`, use `['*']` for all) |
| `allow_credentials` | Enables cookies/authentication in cross-origin requests (default `False`) |
| `expose_headers` | List of response headers exposed to the browser (default `[]`) |
| `max_age` | Time (seconds) browser caches CORS preflight response (default `600`) |

> **Restriction rule:** When `allow_credentials=True`, you cannot use `['*']` for origins, methods, or headers — must specify explicitly.

## SQL Databases

Example using **SQLModel** (built on top of SQLAlchemy and Pydantic):

```python
from typing import Annotated
from fastapi import Depends, FastAPI, HTTPException, Query
from sqlmodel import Field, Session, SQLModel, create_engine, select

class HeroBase(SQLModel):
    # Base model shared across all variants (avoids duplication)
    name: str = Field(index=True)
    age: int | None = Field(default=None, index=True)

class Hero(HeroBase, table=True):
    # Actual database table model (maps to SQL table)
    id: int | None = Field(default=None, primary_key=True)
    secret_name: str  # private field stored in DB but not always exposed

class HeroPublic(HeroBase):
    # Public response model (hides sensitive fields like secret_name)
    id: int

class HeroCreate(HeroBase):
    # Input model for creating new records (client sends this)
    secret_name: str

class HeroUpdate(HeroBase):
    # Input model for partial updates (all fields optional)
    name: str | None = None
    age: int | None = None
    secret_name: str | None = None

sqlite_file_name = "database.db"
sqlite_url = f"sqlite:///{sqlite_file_name}"

# Engine = connection to the database (SQLite here)
connect_args = {"check_same_thread": False}
engine = create_engine(sqlite_url, connect_args=connect_args)

def create_db_and_tables():
    # Creates database tables based on SQLModel classes
    SQLModel.metadata.create_all(engine)

def get_session():
    # Provides a DB session per request (used for CRUD operations)
    with Session(engine) as session:
        yield session

# Dependency injection for DB session
SessionDep = Annotated[Session, Depends(get_session)]

app = FastAPI()

@app.on_event("startup")
def on_startup():
    # Initialize database tables at app startup
    create_db_and_tables()

@app.post("/heroes/", response_model=HeroPublic)
def create_hero(hero: HeroCreate, session: SessionDep):
    # CREATE operation: convert input → DB model
    db_hero = Hero.model_validate(hero)
    session.add(db_hero)          # add new record
    session.commit()              # persist to database
    session.refresh(db_hero)      # refresh with DB-generated values (e.g., id)
    return db_hero                # returned as HeroPublic (no secret_name)

@app.get("/heroes/", response_model=list[HeroPublic])
def read_heroes(
    session: SessionDep,
    offset: int = 0,
    limit: Annotated[int, Query(le=100)] = 100,
):
    # READ operation: fetch multiple records with pagination
    heroes = session.exec(select(Hero).offset(offset).limit(limit)).all()
    return heroes

@app.get("/heroes/{hero_id}", response_model=HeroPublic)
def read_hero(hero_id: int, session: SessionDep):
    # READ operation: get single record by primary key
    hero = session.get(Hero, hero_id)
    if not hero:
        raise HTTPException(status_code=404, detail="Hero not found")
    return hero

@app.patch("/heroes/{hero_id}", response_model=HeroPublic)
def update_hero(hero_id: int, hero: HeroUpdate, session: SessionDep):
    # UPDATE operation: fetch existing record
    hero_db = session.get(Hero, hero_id)
    if not hero_db:
        raise HTTPException(status_code=404, detail="Hero not found")
    # Extract only provided fields (for partial update)
    hero_data = hero.model_dump(exclude_unset=True)
    hero_db.sqlmodel_update(hero_data)  # apply updates to DB object
    session.add(hero_db)     # mark as updated
    session.commit()         # persist changes
    session.refresh(hero_db) # reload updated record
    return hero_db

@app.delete("/heroes/{hero_id}")
def delete_hero(hero_id: int, session: SessionDep):
    # DELETE operation: fetch record
    hero = session.get(Hero, hero_id)
    if not hero:
        raise HTTPException(status_code=404, detail="Hero not found")
    session.delete(hero)  # remove record
    session.commit()      # persist deletion
    return {"ok": True}
```
